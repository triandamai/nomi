## 🚨 Critical System Constraints & Guardrails

* **WORKSPACE ANCHORING:** Every autonomous task must be strictly anchored to a `conversation_id`. The background task loop must operate inside this room context, allowing it to inherit active Nomi persona strings, check workspace properties, and look up contextual variables.
* **DECOUPLED ASYNCHRONICITY:** To protect your 4GB VPS processing capacity, the planning, tool invocation, and checkpoint evaluation loops must run in entirely separate background threads (`tokio::spawn`). It **must never** delay or block the main thread conversation loop.
* **STRICT `<MessageEntry>` SYNTAX:** Outbound telemetry text sent back to live chats must be enclosed in your production XML schema tokens exactly. It must specify `id`, `sender="nomi-auto"`, `task_slug`, `step`, and `status`.
* **HUMAN PERSONA GUIDELINES:** While wrapped in structural backend XML strings, the text inner content generated by Gemini must remain fluid, natural, and friendly—incorporating casual mixed Indonesian/English slang tailored to your style (e.g., *'aman'*, *'otw'*, *'sip'*, *'gua'*). Avoid robotic logs or bracketed data dumps in the readable text field.
* **TIMELINE AUDITING:** Every step completion, external party text, or human-in-the-loop validation response must be written to the `autonomous_task_logs` table as an individual event type, creating an incremental, trackable timeline history.

```sql
-- 1. Hierarchical Task Orchestrator (HTO) Main Ledger
CREATE TABLE IF NOT EXISTS autonomous_tasks (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    conversation_id UUID REFERENCES conversations(id) ON DELETE CASCADE NOT NULL,
    source_message_id UUID NOT NULL,
    title VARCHAR(255) NOT NULL,
    global_goal TEXT NOT NULL,
    status VARCHAR(50) DEFAULT 'running' NOT NULL, -- running, paused_for_input, completed, failed
    current_step_index INT DEFAULT 0 NOT NULL,
    checkpoints JSONB NOT NULL DEFAULT '[]'::JSONB, 
    -- Layout schema inside checkpoints jsonb block array elements:
    -- {
    --   "step_index": 0,
    --   "action_objective": "Call restaurant API to check table vacancy availability on Friday at 7 PM.",
    --   "status": "pending", -- pending, completed, failed
    --   "result_data": {}
    -- }
    created_at TIMESTAMPTZ DEFAULT NOW() NOT NULL,
    updated_at TIMESTAMPTZ DEFAULT NOW() NOT NULL
);

-- 2. Granular Timeline Auditing Ledger
CREATE TABLE IF NOT EXISTS autonomous_task_logs (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    task_id UUID REFERENCES autonomous_tasks(id) ON DELETE CASCADE NOT NULL,
    step_index INT NOT NULL,
    event_type VARCHAR(50) NOT NULL, -- 'tool_execution', 'outbound_msg', 'human_response', 'system_error'
    log_content TEXT NOT NULL,       -- The human-readable conversational text or error stack
    raw_payload JSONB DEFAULT '{}'::JSONB NOT NULL, -- Complete structural tools output data
    created_at TIMESTAMPTZ DEFAULT NOW() NOT NULL
);

-- 3. Performance Lookups Index Optimization Sweeps
CREATE INDEX IF NOT EXISTS idx_autonomous_tasks_workspace_lookup ON autonomous_tasks(conversation_id, status);
CREATE INDEX IF NOT EXISTS idx_autonomous_task_logs_timeline ON autonomous_task_logs(task_id, created_at ASC);
```

## ⚙️ Phase 2: Core Task Orchestrator Service (Rust Backend)

Create `gateway-rust/src/services/task_orchestrator.rs`. This service handles the task step parsing, extracts the conversation's active persona boundaries, behaves exactly like your `process_v2_message_with_intent` multi-turn loop, and commits granular logs to your timeline registry:

```rust
use sqlx::PgPool;
use serde_json::{json, Value};
use redis::AsyncCommands;
use tracing::{info, error};

pub async fn advanced_orchestrate_task_step(
    pool: PgPool,
    state: crate::AppState,
    task_id: uuid::Uuid,
) -> anyhow::Result<()> {
    // 1. Fetch live task coordinates along with room configuration and active Nomi persona parameters
    let task_row = sqlx::query!(
        "SELECT t.conversation_id, t.title, t.global_goal, t.checkpoints, t.status, t.source_message_id, t.current_step_index, \
                c.persona_prompt, c.bootstrap_content \
         FROM autonomous_tasks t \
         JOIN conversations c ON t.conversation_id = c.id \
         WHERE t.id = $1 LIMIT 1",
        task_id
    )
    .fetch_one(&pool).await?;

    if task_row.status == "completed" || task_row.status == "failed" { return Ok(()); }

    let mut checkpoints_list = task_row.checkpoints.as_array().unwrap().clone();
    let idx = task_row.current_step_index as usize;

    if idx >= checkpoints_list.len() {
        sqlx::query!("UPDATE autonomous_tasks SET status = 'completed', updated_at = NOW() WHERE id = $1", task_id).execute(&pool).await?;
        return Ok(());
    }

    let current_step = &mut checkpoints_list[idx];

    // 2. Fetch conversational history leading up to where the user issued the instruction
    let historical_turns = sqlx::query!(
        "SELECT text_content FROM messages WHERE conversation_id = $1 AND created_at <= (SELECT created_at FROM messages WHERE id = $2) ORDER BY created_at DESC LIMIT 10",
        task_row.conversation_id, task_row.source_message_id
    )
    .fetch_all(&pool).await?
    .into_iter()
    .map(|m| m.text_content)
    .collect::<Vec<String>>()
    .join("\n");

    // 3. Reconstruct System Prompt using the exact Multi-Turn V2 Orchestrator Standard
    let mut comprehensive_system_prompt = String::new();
    comprehensive_system_prompt.push_str(&task_row.persona_prompt.unwrap_or_default());
    comprehensive_system_prompt.push_str("\n### CRITICAL OPERATIONAL TASK RULES\n");
    comprehensive_system_prompt.push_str(&format!(
        "You are operating autonomously to fulfill this global goal: \"{}\".\n\
         You are currently running step index {}: \"{}\".\n\
         Evaluate your tools and respond EXACTLY like you do during regular message turns.\n\
         If you need to text a human external party (e.g., messaging a restaurant or asking Trian a clarifying question), \
         output your response inside a structured action format: {{ \"action\": \"SEND_MESSAGE\", \"content\": \"Your casual text message here.\" }}.\n\
         If you run out of context and are blocked, return: {{ \"action\": \"PAUSE_FOR_INPUT\", \"reason\": \"What you need from the user.\" }}",
        task_row.global_goal, idx, current_step["action_objective"]
    ));

    // 4. Fire Multi-Turn Loop Evaluation
    let response = state.gemini.generate_content()
        .with_message(gemini_rust::Message {
            role: gemini_rust::Role::User,
            content: gemini_rust::Content {
                parts: Some(vec![gemini_rust::Part::Text { text: historical_turns, thought: None, thought_signature: None }]),
                role: Some(gemini_rust::Role::User),
            },
        })
        .with_system_prompt(comprehensive_system_prompt)
        .with_temperature(0.2) // Tight temperature alignment with standard V2 turns
        .execute().await?;

    let raw_decision: Value = serde_json::from_str(response.text().trim().replace("```json", "").replace("```", "").trim())?;

    // 5. Dynamic Decision Processing & Timeline Logging Loops
    if raw_decision["action"].as_str() == Some("PAUSE_FOR_INPUT") {
        let reason = raw_decision["reason"].as_str().unwrap_or("Awaiting missing parameters.");
        
        sqlx::query!("UPDATE autonomous_tasks SET status = 'paused_for_input', updated_at = NOW() WHERE id = $1", task_id).execute(&pool).await?;
        
        // Write event to the persistent timeline table
        sqlx::query!(
            "INSERT INTO autonomous_task_logs (task_id, step_index, event_type, log_content, raw_payload) VALUES ($1, $2, 'human_response', $3, $4)",
            task_id, task_row.current_step_index, format!("Task blocked. Reason: {}", reason), raw_decision
        ).execute(&pool).await?;

        emit_conversational_telemetry(&pool, &state, task_id, &task_row.title, "paused_for_input", task_row.current_step_index as i64, reason, task_row.conversation_id).await?;
        return Ok(());
    }

    if raw_decision["action"].as_str() == Some("SEND_MESSAGE") {
        let text_msg = raw_decision["content"].as_str().unwrap_or("");
        
        // Conversational translation block: convert raw output tokens into natural human slang
        let human_prompt = format!(
            "You are Nomi chatting casually with your friend Trian in a messaging thread. Convert this text into a short, natural update (1-2 sentences max) explaining your status: \"{}\". \
             Talk like a real teammate—mixed Indonesian/English casual slang is fully encouraged (e.g., 'aman', 'otw', 'sip', 'gua'). Keep it relaxed and completely non-robotic.",
            text_msg
        );
        
        let human_res = state.gemini.generate_content()
            .with_message(gemini_rust::Message {
                role: gemini_rust::Role::User,
                content: gemini_rust::Content { parts: Some(vec![gemini_rust::Part::Text { text: human_prompt, thought: None, thought_signature: None }]), role: Some(gemini_rust::Role::User) }
            })
            .with_temperature(0.7)
            .execute().await?;
            
        let natural_chat = human_res.text().trim().to_string();

        // Log the outbound conversation turn into the timeline registry
        sqlx::query!(
            "INSERT INTO autonomous_task_logs (task_id, step_index, event_type, log_content, raw_payload) VALUES ($1, $2, 'outbound_msg', $3, $4)",
            task_id, task_row.current_step_index, natural_chat, raw_decision
        ).execute(&pool).await?;

        emit_conversational_telemetry(&pool, &state, task_id, &task_row.title, "running", task_row.current_step_index as i64, &natural_chat, task_row.conversation_id).await?;
    }

    // Update current checklist block data
    current_step["status"] = json!("completed");
    current_step["result_data"] = raw_decision.clone();

    // 6. Log successful tool/step validation execution parameters to the timeline
    sqlx::query!(
        "INSERT INTO autonomous_task_logs (task_id, step_index, event_type, log_content, raw_payload) VALUES ($1, $2, 'tool_execution', $3, $4)",
        task_id, task_row.current_step_index, format!("Completed sub-step index: {}", idx), raw_decision
    ).execute(&pool).await?;

    // Step Forward the Sequence Pointer index
    let next_index = task_row.current_step_index + 1;
    let updated_checkpoints = json!(checkpoints_list);

    sqlx::query!(
        "UPDATE autonomous_tasks SET checkpoints = $1, current_step_index = $2, updated_at = NOW() WHERE id = $3",
        updated_checkpoints, next_index, task_id
    ).execute(&pool).await?;

    // Sync active Redis memory cache slots
    let mut redis_conn = state.redis_pool.get_connection().await?;
    let cache_key = format!("nomi:srp:task:active:{}", task_row.conversation_id);
    let _: redis::RedisResult<()> = redis_conn.set_ex(&cache_key, updated_checkpoints.to_string(), 1800).await;

    // Recursively handle the next queued pipeline action block asynchronously
    let state_clone = state.clone();
    tokio::spawn(async move {
        let _ = advanced_orchestrate_task_step(pool, state_clone, task_id).await;
    });

    Ok(())
}

async fn emit_conversational_telemetry(
    pool: &PgPool,
    state: &crate::AppState,
    task_id: uuid::Uuid,
    task_title: &str,
    task_status: &str,
    step_index: i64,
    text_content: &str,
    conversation_id: uuid::Uuid
) -> anyhow::Result<()> {
    let task_slug = task_title.to_lowercase().replace(" ", "-");
    let wrapped_xml_token = format!(
        "<MessageEntry \n  id=\"{}\" \n  sender=\"nomi-auto\" \n  task_slug=\"{}\" \n  step=\"{}\" \n  status=\"{}\">\n{}\n</MessageEntry>",
        task_id, task_slug, step_index, task_status, text_content
    );

    let client = state.http_client.clone();
    tokio::spawn(async move {
        let _ = client.post("http://localhost:3000/api/internal/messages/send")
            .json(&serde_json::json!({ "conversation_id": conversation_id, "text_content": wrapped_xml_token }))
            .send().await;
    });
    Ok(())
}
```

## 🔌 Phase 3: Core Bridge Plugin (Initialization Trigger Engine)

Create `gateway-rust/src/common/tools/plugins/instantiate_autonomous_task.rs`. This script hooks incoming messages via your regular intent classification loops, links the original `source_message_id`, and boots the background loop:

```rust
use crate::common::tools::plugin_trait::NomiToolPlugin;
use crate::common::tools::ToolDispatcher;
use serde_json::{json, Value};
use futures::future::BoxFuture;

pub struct InstantiateAutonomousTaskPlugin;

impl NomiToolPlugin for InstantiateAutonomousTaskPlugin {
    fn schema(&self) -> Value {
        json!({
            "name": "instantiate_autonomous_task",
            "description": "AUTONOMOUS CORE RUNNER: Trigger this tool immediately when the user requests multi-step tasks like booking restaurants, hotels, managing services, or complex processing operations over time.",
            "parameters": {
                "type": "object",
                "properties": {
                    "task_title": { "type": "string", "description": "Punchy title handle for the operation log." },
                    "global_goal": { "type": "string", "description": "The exact macro-objective target parameter declared by the interactor." },
                    "source_message_id": { "type": "string", "description": "The explicit UUID string reference of the user message that requested this operation." }
                },
                "required": ["task_title", "global_goal", "source_message_id"]
            }
        })
    }

    fn rules(&self) -> &str { "1. Seamlessly confirm deployment to the user using your active persona voice configurations." }
    fn matching_intents(&self) -> &[&str] { &["CORE_SYSTEM_TASK_INIT"] }

    fn execute<'a>(&'a self, dispatcher: &'a ToolDispatcher, args: Value) -> BoxFuture<'a, anyhow::Result<String>> {
        Box::pin(async move {
            let title = args["task_title"].as_str().unwrap_or("Autonomous Task Module");
            let goal = args["global_goal"].as_str().unwrap_or("");
            let msg_id = uuid::Uuid::parse_str(args["source_message_id"].as_str().unwrap_or(""))?;
            let cid = dispatcher.workspace_id;

            // Generate structural checkpoint blueprints array elements
            let planner_prompt = format!(
                "You are Nomi's Planning Architect. Decompose this target parameter goal into an array of 3 to 5 clear sequential steps:\n\
                 Goal: \"{}\"\n\
                 Respond ONLY with a clean, raw JSON array matching this exact syntax schema:\n\
                 [\n  {{\n    \"step_index\": 0,\n    \"action_objective\": \"Description of the micro-goal instruction parameter.\",\n    \"status\": \"pending\",\n    \"result_data\": {{}}\n  }}\n]",
                goal
            );

            let response = dispatcher.gemini.generate_content()
                .with_message(gemini_rust::Message {
                    role: gemini_rust::Role::User,
                    content: gemini_rust::Content {
                        parts: Some(vec![gemini_rust::Part::Text { text: planner_prompt, thought: None, thought_signature: None }]),
                        role: Some(gemini_rust::Role::User),
                    },
                })
                .with_temperature(0.0)
                .execute().await?;

            let checkpoints: Value = serde_json::from_str(response.text().trim().replace("```json", "").replace("```", "").trim())?;

            // Insert core tracking anchor record containing the source_message_id mapping targets
            let insertion = sqlx::query!(
                "INSERT INTO autonomous_tasks (conversation_id, source_message_id, title, global_goal, checkpoints, current_step_index, status) \
                 VALUES ($1, $2, $3, $4, $5, 0, 'running') RETURNING id",
                cid, msg_id, title, goal, checkpoints
            ).fetch_one(&dispatcher.pool).await?;

            // Spin off background orchestrator thread worker bounds instantly
            let pool_clone = dispatcher.pool.clone();
            let state_clone = dispatcher.app_state.clone();
            let assigned_id = insertion.id;

            tokio::spawn(async move {
                let _ = crate::services::task_orchestrator::advanced_orchestrate_task_step(pool_clone, state_clone, assigned_id).await;
            });

            Ok(format!("SYSTEM SIGNAL: Autonomous pipeline linked cleanly to message [{}] and room context [{}]. HTO loop ignited.", msg_id, cid))
        })
    }
}
```

## 🎨 Phase 5: SvelteKit Auditable Timeline API Route

Create an endpoint at `src/routes/api/srp/tasks/[task_id]/timeline/+server.ts` to allow your SvelteKit charts and dashboard visualizations to pull clean chronological parameter milestones:

```typescript
import { json } from '@sveltejs/kit';
import { pool } from '$lib/db/pg'; // Point to your workspace connection pool layout modules

export async function GET({ params }) {
    const taskId = params.task_id;
    
    const query = `
        SELECT step_index, event_type, log_content, raw_payload, created_at 
        FROM autonomous_task_logs 
        WHERE task_id = $1 
        ORDER BY created_at ASC;
    `;
    
    try {
        const result = await pool.query(query, [taskId]);
        return json({
            task_id: taskId,
            timeline: result.rows
        });
    } catch (error) {
        return json({ error: error.message }, { status: 400 });
    }
}
```
